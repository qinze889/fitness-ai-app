# Review Process

Use this review loop for every Codex implementation task:

1. Check task scope.
2. Check diff.
3. Check build result.
4. Check APK path if relevant.
5. Check acceptance criteria.
6. List required fixes.
7. Approve only after the task is buildable and within scope.

Review order:

Spec -> Task Breakdown -> Diff -> Build Result -> APK Result -> Acceptance -> Repair Notes

If a task fails review, send the repair notes back to Codex and ask it to fix the same branch only.
